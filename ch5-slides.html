Chapter 5 · SQL
SQL — Talking to a Database
Structured Query Language — the universal language for asking questions of relational databases
-- Find all customers from Miami who spent over $500
SELECT c.Name, c.Email, o.Product, o.Amount
FROM Customers c
JOIN Orders o ON c.CustomerID = o.CustomerID
WHERE c.City = 'Miami'
  AND o.Amount > 500
ORDER BY o.Amount DESC;

Scenario 1
A store needs to find all customers who live in Florida AND bought a product in the last 30 days. What SQL keyword filters the results?
WHERE — filters rows based on conditions. The query would include: WHERE State = 'Florida' AND PurchaseDate >= [30 days ago]. AND combines multiple conditions.

Scenario 2
A hospital stores patients in one table and appointments in another. The same PatientID appears in both. What is PatientID called in the appointments table?
Foreign Key — it references the Primary Key in the Patients table. This relationship allows the database to connect a patient's profile to all their appointments without duplicating patient data.

Scenario 3
A manager asks: "Which of our products has generated the most revenue this quarter?" What type of system manages the underlying data and handles this query?
DBMS (Database Management System) — software like Oracle, MySQL, or SQL Server that manages the database, handles queries, enforces security, and ensures data integrity. The manager's question is answered via a SQL SELECT query.

Scenario 4
In a STUDENTS table, two students have the same name "Michael Smith." How does the database tell them apart?
Primary Key — a unique identifier (like StudentID: S001 vs S002) that distinguishes every row. A Primary Key can NEVER be null or duplicate. This is why we don't use names as primary keys.

Chapter 5 · Big Data
Big Data: The 5 Vs
Traditional databases can't handle it. Click each V to see what makes Big Data different.
V
Volume
Massive scale — terabytes, petabytes, exabytes. Too large for traditional systems.
Facebook stores 100+ petabytes of photos. Walmart processes 1M+ transactions/hour.
V
Velocity
Real-time or near-real-time data streams. Must be processed instantly as it arrives.
Twitter processes 500M+ tweets/day. Stock exchanges process millions of trades/second.
V
Variety
Many data types: structured (tables), unstructured (text, video, audio, images, sensor data).
A hospital collects lab results (structured), MRI scans (images), and doctors' notes (unstructured text).
V
Veracity
Trustworthiness and accuracy of the data. Big data is often messy, incomplete, or contradictory.
Social media data is noisy — bots, duplicates, sarcasm — cleaning it is a major challenge.
V
Value
The ultimate goal — extracting business value from the data. Big data is worthless without analysis.
Netflix's recommendation engine (built on Big Data) is estimated to save $1B/year in subscriber retention.
Exam Note The exam most commonly tests the original 3 Vs: Volume, Velocity, Variety. Know all five but prioritize those three.

Chapter 5 · Data Architecture
Database vs. Data Warehouse
They store data — but for completely different purposes. This distinction is a guaranteed exam topic.
Feature 📋 Operational Database 🏛️ Data Warehouse
Purpose Run the business day-to-day (OLTP) Analyze the business over time (OLAP)
Data Age Current — updated in real time Historical — weeks, months, years of data
Query Type Simple — find one order, update one record Complex — "revenue by region for last 3 years"
Users Clerks, cashiers, customer service reps Analysts, executives, data scientists
Example Your bank processing a $20 ATM withdrawal Bank analyzing fraud patterns across all ATMs nationwide
Exam Trap A data warehouse is NOT for real-time transactions. It's for historical analysis. Confusing these two is the most common Ch 5 exam mistake.

Chapter 5 · Takeaways
What You Must Know
  • Data → Information → Knowledge → Wisdom: IS transforms raw facts into actionable decisions. Data alone has no value.
  • Relational database: data in tables (rows = records, columns = fields), linked by Primary Keys (unique row ID) and Foreign Keys (link to another table).
  • SQL = Structured Query Language — SELECT, FROM, WHERE, JOIN are the core commands. You don't write it, but you must understand what it does.
  • DBMS (Oracle, MySQL, SQL Server) manages database creation, querying, security, and integrity. It's the software layer above the data.
  • Big Data 3 Vs (exam priority): Volume (massive scale) · Velocity (real-time speed) · Variety (structured + unstructured). Also: Veracity, Value.
  • Data Warehouse ≠ Database: warehouses are for historical analysis (OLAP), not live transactions (OLTP). Data mining discovers hidden patterns in large datasets.
